# SPARR Architecture

## Current Architecture

SPARR transforms user ideas into structured intellectual challenge.

```text
User Idea
    ↓
Conversation Engine
    ↓
Perspective Engine
    ↓
Challenge Engine
    ↓
AI Model
    ↓
Reflection Layer
    ↓
Response
```

## Frontend Layer

Technology:

* React
* TypeScript
* Tailwind CSS

Responsibilities:

* User Interface
* Chat Experience
* Voice Experience
* Perspective Selection
* Session History

## Conversation Engine

Purpose:

Manage natural conversation flow.

Responsibilities:

* Understand user intent
* Maintain context
* Adapt language complexity
* Keep conversations human

Output:

Conversation-ready context.

## Perspective Engine

Purpose:

Determine how an idea should be challenged.

Perspectives:

* Scientist
* Philosopher
* Historian
* Economist
* Skeptic
* Founder

Responsibilities:

* Select perspective
* Combine perspectives
* Route challenge generation

Output:

Challenge strategy.

## Challenge Engine

Purpose:

Generate intellectual pressure.

Responsibilities:

* Select challenge pattern
* Generate questions
* Escalate difficulty
* Detect weak reasoning

Output:

Challenge prompts.

## AI Model Layer

Current:

* Gemini API

Future:

* Multi-model support
* Model comparison
* Specialized reasoning models

Responsibilities:

* Generate responses
* Maintain coherence
* Follow SPARR behavior rules

## Reflection Layer

Purpose:

Turn conversations into learning.

Responsibilities:

* Detect blind spots
* Identify assumptions
* Highlight strengths
* Summarize insights

Output:

Actionable reflection.

## Core Technical Challenge

The hardest problem is not generating arguments.

The hardest problem is creating conversations that feel:

* natural
* useful
* understandable
* challenging
* human

SPARR succeeds when users feel understood before they feel challenged.
